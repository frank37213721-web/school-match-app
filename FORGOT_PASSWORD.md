# 忘記密碼功能說明

## 🔐 功能概述

為學校帳號登入提供安全的密碼重置功能，透過多重身分驗證確保安全性。

## 🎯 功能特色

### 🔍 多重身分驗證
1. **帳號驗證** - 學校電話號碼
2. **姓名驗證** - 承辦人姓名
3. **Email 驗證** - 承辦人 Email

### 🔄 安全重置流程
- 密碼重置為預設密碼（電話號碼後4碼）
- 自動雜湊處理
- Email 通知所有相關人員
- 操作日誌記錄

### 📧 多層次通知
- **承辦人** - 詳細的重置資訊和新密碼
- **承辦處室主任** - 重置操作通知
- **校長** - 重置操作通知

## 📋 使用流程

### 1. 用戶操作
```
1. 點擊「忘記密碼」區域
2. 輸入學校帳號（電話號碼）
3. 輸入承辦人姓名
4. 輸入承辦人 Email
5. 點擊「🔄 重置密碼」
```

### 2. 系統驗證
```
✅ 帳號存在性檢查
✅ 承辦人姓名比對
✅ 承辦人 Email 比對
❌ 任一驗證失敗則顯示具體錯誤訊息
```

### 3. 重置執行
```
✅ 生成預設密碼（電話後4碼）
✅ 雜湊處理新密碼
✅ 更新資料庫
✅ 發送通知 Email
```

## 🛡️ 安全機制

### 身分驗證
- **三要素驗證** - 帳號、姓名、Email 必須全部正確
- **精確比對** - 區分大小寫和空格
- **錯誤提示** - 不透露具體哪個欄位錯誤

### 密碼安全
- **預設密碼** - 使用電話號碼後4碼，易記憶但相對安全
- **雜湊處理** - 自動進行 bcrypt 雜湊
- **建議修改** - 提醒用戶登入後修改密碼

### 通知機制
- **即時通知** - 重置後立即發送 Email
- **多層通知** - 通知承辦人、承辦處室主任、校長
- **操作記錄** - 包含重置時間戳記

## 🔧 技術實現

### 核心流程
```python
# 1. 查詢學校資料
school = supabase.table("schools").select("*").eq("phone", phone).execute()

# 2. 多重驗證
if school['registrant_name'] != name: fail()
if school['registrant_email'] != email: fail()

# 3. 重置密碼
default_password = phone[-4:]
hashed_password = hash_password(default_password)
supabase.table("schools").update({"password_hash": hashed_password}).execute()

# 4. 發送通知
send_password_reset_email(school, default_password)
```
